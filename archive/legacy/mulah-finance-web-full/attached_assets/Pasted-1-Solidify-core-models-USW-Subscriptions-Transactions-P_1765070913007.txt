2. Subscription detection & linkage

We already have subscription tracking, but I want a clear subscription detection and linkage flow:

Implement or confirm a simple recurring detection process that:

Groups transactions with:

same merchant or similar description

similar amounts

roughly consistent intervals (e.g. 28–32 days etc.)

For each detected recurring group, either:

Link it to an existing Subscription, or

Offer / create a new Subscription record.

Expose an endpoint such as:

POST /api/subscriptions/detect – runs detection for a user and returns a list of suggested subscriptions with confidence scores.

4. CFA v0 – Insights service

Introduce a basic Complex Finance Analyzer service:

Create a service/module, e.g. cfaService, that:

Takes a user (and optional time range).

Reads all relevant transactions (across accounts).

Returns a small object of high-impact insights, for example:

average monthly surplus/deficit over last 6 months

top 3 categories by spend

top 3 subscriptions by cost

one or two risk signals (e.g. “x% of your income is locked in fixed subscriptions”)

Expose an endpoint such as:

GET /api/cfa/summary?from=&to=

Frontend:

Add a “Insights” section which calls this endpoint and displays the insights in a simple card layout.

Design the CFA service so it’s easy to extend later with more complex analysis.
